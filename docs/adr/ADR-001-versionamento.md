# ADR-001: Adoção do GitHub Flow

## Status

Aceito

## Contexto

A equipe necessita de uma estratégia de versionamento para organizar o desenvolvimento colaborativo e controlar alterações no código-fonte.

## Alternativas Consideradas

- Git Flow
- GitHub Flow

## Decisão

Foi decidido adotar o GitHub Flow.

## Justificativa

A estratégia é simples, adequada para equipes pequenas e integra-se naturalmente ao uso de Pull Requests.

## Consequências

### Positivas

- Fluxo simples de desenvolvimento.
- Facilita revisões de código.
- Menor esforço de gerenciamento.

### Negativas

- Menos adequada para manutenção simultânea de múltiplas versões.
